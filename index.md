## 💡 Professional Reflections (STAR Method)

### 1. Reflection on Coding in Markdown
* **Situation:** For my professional portfolio, I needed a way to document my technical skills and projects clearly without the complexity of full HTML/CSS for every text block.
* **Task:** My goal was to create a clean, readable CV and project descriptions that would render perfectly on GitHub.
* **Action:** I utilized GitHub Flavored Markdown to structure my headers, tech stack lists, and project links. I focused on using proper syntax like backticks for code and clear hierarchies for information.
* **Result:** This resulted in a professional layout that is easy to maintain. I learned that Markdown is an essential tool for developer communication, ensuring documentation is as organized as the code itself.

### 2. Reflection on Mock Interview Experience
* **Situation:** As part of my work-readiness training, I participated in a recorded mock interview to simulate a real-world software engineering hiring process.
* **Task:** I had to articulate my technical decisions and problem-solving process—specifically regarding my web development projects—while maintaining a professional demeanor.
* **Action:** During the interview, I focused on my communication speed. I noticed I tend to talk faster when explaining logic, so I consciously slowed down to ensure the interviewer could follow my thought process.
* **Result:** Reviewing the video helped me identify that while my technical knowledge is strong, I need to keep my explanations concise. It was a vital step in moving from a student mindset to a professional one.

### 3. Reflection on Use of GitHub Pages
* **Situation:** Once my portfolio was designed, I needed a reliable way to host it so it could be accessed by lecturers and potential employers via a live URL.
* **Task:** The objective was to deploy my repository to GitHub Pages to create a public-facing version of my digital portfolio.
* **Action:** I configured the repository settings to deploy from the main branch and optimized my file structure to ensure all assets, like my CV and video, loaded correctly.
* **Result:** Successfully launching this site gave me a centralized, professional hub. This taught me the importance of the deployment phase in the software lifecycle and gave me the confidence to share my work publicly.
